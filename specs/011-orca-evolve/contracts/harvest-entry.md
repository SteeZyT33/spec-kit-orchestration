# Contract: Harvest Entry

## Purpose

Define the minimum durable record for one external idea under Orca adoption
review.

## Required Properties

- source attribution
- concise summary
- explicit decision
- rationale
- explicit Orca target mapping or explicit absence of one
- durable timestamps
- explicit entry kind so wrapper capabilities can be distinguished from general
  harvested patterns
- explicit adoption scope so portable principles can be separated from
  host-specific implementation detail

## V1 Rules

- each entry represents one idea/pattern
- entries must remain independently inspectable
- copy-paste adoption is allowed only with explicit attribution and mapping
- if an entry represents a thin Orca wrapper over an external specialist
  system, the record must state the external dependency and ownership boundary
- if an entry comes from a provider- or runtime-specific system, the record
  must state whether Orca is adopting the portable principle, the host-specific
  detail, or intentionally rejecting the host-specific parts
