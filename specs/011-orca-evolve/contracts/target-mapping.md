# Contract: Target Mapping

## Purpose

Define how harvested ideas point into Orca's real upgrade system.

## Allowed V1 Targets

- existing Orca spec
- future feature slot
- capability pack
- roadmap placeholder
- none

## V1 Rules

- every non-rejected entry must either map to a target or be explicitly deferred
- target mapping must be durable and reviewable
- mappings must not replace the destination spec; they only link into it
