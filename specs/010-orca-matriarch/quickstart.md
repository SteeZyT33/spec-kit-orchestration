# Quickstart: Orca Matriarch

## Goal

Use one supervisory interface to manage several active feature lanes safely.

## Example Flow

1. Register or create lanes for active specs.
2. Declare dependencies between lanes where needed.
3. Attach or record worktree/branch targets for each lane.
4. Assign owners or agents to each lane.
5. Inspect the Matriarch status view to see active, blocked, review-ready, and
   PR-ready lanes.
6. Resolve checkout targets when moving into a lane.
7. Revisit the status view to see readiness and next actions derived from
   durable artifacts.

## Expected Operator Value

- one place to inspect multiple specs
- fewer branch/worktree mistakes
- explicit blocked/ready visibility
- less manual coordination overhead without surrendering control
